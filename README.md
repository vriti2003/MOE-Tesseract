# MOE-Tesseract
## Beyond Monolithic OCR: Mixture-of-Experts Specialisation for Ancient Devanagari Script Recognition

Recognizing ancient Devanagari manuscripts remains an open challenge in document image analysis due to conjunct-heavy glyph sequences, ink bleed, clutter, and severe page degradation, which lead to the failure of OCR systems trained primarily on modern corpora. There has been limited progress constrained as much by data scarcity as by modelling limitations. As a first contribution, we release **Extended-AnciDev**, scaling the AnciDev dataset from 500 to 3{,}000 annotated pages of ancient Devanagari manuscripts, broadening coverage across manuscript sources, domains, time periods, and degradation conditions. The widely used OCR model, Tesseract-5, uses an LSTM-based engine that processes each input line with a single shared parameter set, allocating identical representational capacity to all scripts, fonts, and degradations, regardless of their complexity or frequency. This monolithic design creates a fundamental uncertainty: increasing model capacity benefits all scripts uniformly, while practical deployment demands per-script specialization without runtime language pre-selection. As a second contribution, we present **MoE-Tesseract**, the first integration of Mixture-of-Experts~(MoE) into the Tesseract~5 recognition pipeline, to the best of our knowledge. MoE-Tesseract routes each input line to a sparse subset of expert networks, enabling script-aware specialization without architectural redesign. Trained entirely on CPU via Tesseract's existing OpenMP pipeline, **MoE-Tesseract** reduces **CER** by **7.51%** and **WER** by **8.40%** on AnciDev relative to the **fine-tuned Tesseract-5** baseline, and **CER** by **15.21%** and **WER** by **4.67%** on **Extended-AnciDev**. These results demonstrate that conditional computation offers a practical path toward specialized historical script recognition within the constraints of open-source OCR infrastructure.
